## Key decisions made this session

1. **Orientation-homogeneous rule:** Each reel and story is entirely vertical OR entirely horizontal, never mixed. Posts are always horizontal/square.
2. **Max-2x reuse rule:** Each photo may be used up to 2 times across the 14-piece package, but never twice within the same piece. Replaces the old "each photo used at most once" rule.
3. **New upload limits:** Horizontal photos: min 20, max 40 (was 10/30). Vertical photos: 0 or min 5, max 40.
4. **Vertical hero concept:** Stored as `vertical_hero_photo_id` on listings table. Nullable. Defaults to first vertical photo by sort_order if not explicitly set. Used as slot 1 in all-vertical Day 2 reels.
5. **Server-side validation only for verticals:** The upload UI shows warnings but does not block submission. Validation enforced in `submitListingForReview` and `/api/checkout`. Reason: orientation is detected asynchronously by the fire-and-forget `/api/photos/process` route, so client-side blocking would be unreliable.
6. **Two-grid UI:** Horizontal photos in main grid (with HERO badge), vertical photos in a separate grid below (with V-HERO badge). Both grids share the same underlying listing_photos table.
7. **Reel priority over stories:** When verticals are scarce, reels consume them first. Processing order: Day 2 reel → remaining reels → stories → posts.
8. **Day 2 reel vertical threshold:** Requires vertical hero + 4 unused pool verticals (total 5 vertical photos) to go all-vertical. Other reels require 5 unused pool verticals.
9. **Deferred to Phase 2:** Template library with random selection per content piece, blur fill template design in Creatomate editor, square post template variants, stories-as-video content mix, drag-reorder batch update optimization.
