# v1.1 — Hybrid Photo System Spec

**Status:** Draft — start implementing next session
**Owner:** Colby
**Depends on:** Creatomate integration (already complete, template day1_just_listed rendering successfully)

## Problem statement

Two issues observed during the first Creatomate test render on 2026-04-10:

1. **Orientation mismatch.** Listing photos are almost always shot horizontal (landscape), but Stories and Reels render vertical (1080x1920). Creatomate center-crops by default, chopping off the left and right sides of every photo. The realtor beta user called this out as the #1 visual problem with the product.

2. **Visual variety.** The first 5 photos by sort_order on a real listing turned out to be 5 consecutive shots from the same camera session of similar subjects (e.g. 3 front exterior angles in a row). Pipeline logic is correct — photos are distinct files — but perceptually the reel looks like it's repeating the same shot.

## Solution architecture — hybrid A+C

### Phase 1 — Orientation (next session)

1. **Database:** add `orientation` column to `listing_photos` table. Values: `"horizontal"` | `"vertical"` | `"square"`. Migration should backfill existing rows as `"horizontal"` default.

2. **Upload flow:** auto-detect orientation on upload using the existing `sharp` dependency. Compute `width / height` ratio: >1.1 = horizontal, <0.9 = vertical, else square. Write to the new column.

3. **Upload UI:** add an OPTIONAL second photo upload section titled "Vertical photos (optional — for Stories & Reels)" with helper text explaining it's optional and that horizontal photos will be used with a blur background fill if none are provided. This section goes BELOW the existing primary photo uploader, not replacing it.

4. **Photo selection logic:** update the pipeline photo picker. For each content piece, check the target format:
   - Horizontal format (1080x1080 posts, 1200x630 FB) → pick from horizontal photos as today
   - Vertical format (1080x1920 Stories & Reels) → prefer vertical photos if any exist; fall back to horizontal if none
   - Square format → either works

5. **Creatomate template rebuild:** rebuild the day1_just_listed template in the Creatomate editor to include a **blur background fill layer**. Pattern: for each Photo-N slot, add a second image element behind the main one, same source URL, stretched to fill the full 1080x1920 frame, with a heavy blur filter (~40px) and a dark overlay (~30% black). The main photo sits on top at its true aspect ratio, letterboxed against the blurred background. This becomes the universal fallback whenever a horizontal photo ends up in a vertical slot.

### Phase 2 — Visual variety (deferred — not next session)

Use Claude Sonnet vision API at upload time to tag each photo with a room/feature label (exterior, kitchen, living_room, primary_bedroom, bathroom, outdoor, pool, detail_shot, etc.). Store in a new `photo_tags` column on `listing_photos`. Photo picker enforces variety: no two consecutive reel slots get the same tag. Cost estimate: ~$0.01 per photo × 20 photos = $0.20 per listing upload. Implement after Phase 1 ships and we see whether the orientation fix alone is enough to satisfy the realtor.
